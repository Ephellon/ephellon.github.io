Paramour is a client-side, preprocessed JavaScript (JS) language, based off of CoffeeScript (CS), Java, Python, Swift, and new ECMAScript features. Though JavaScript is powerful alone, Paramour enhances and simplifies JavaScript's best and worst qualities.

It is assumed that you, the reader, know at least the basics of JavaScript, and if not, see "JavaScript" by W3Schools.

(Get ready for a headache) Paramour is a multi(16)-paradigm language, it can be generalized as a:
Dynamically Imperative, Event Driven, Object, Prototype, Expression, and Protocol Oriented, Block Structured, End-User Scripting language.

Paramour's Programming Paradigms

  • Block Structured
  • Conceptual

"text/paramour" Script Tags

If you want to run a Paramour script that is loaded via a URL, then this section is for you.
To use this feature, you must have a <script type="text/paramour"> tag, or type="text/par" if you feel a bit lazy. You can download the latest version here, about 47.7 Kb.
Afterwards, you simply call Paramour.compile() and Paramour will handle the rest from there.
A simple example: 

<head>
  <script src="par.min.js"></script>
  <script src="Gumball.par" type="text/par"></script>
</head>
<body onload="Paramour.compile()">
  <!-- blah blah blah -->
</body>
